# 《今日约会男友》游戏策划书

> **DATING BOYFRIEND TODAY**
> 一款以"恋爱手账"为概念的轻量级移动端 H5 互动游戏

---

## 一、游戏概述

### 1.1 游戏定位
面向年轻女性用户的**轻休闲 + 叙事驱动**恋爱体验游戏。玩家扮演女主角，在五位性格迥异的男主角之间展开心动约会，通过"收集爱意、躲避伤痛"的核心玩法推进故事，最终点亮每位男主的专属卡牌。

### 1.2 核心概念
- **恋爱手账**：整个游戏以"翻开手账"为叙事隐喻，每一次约会都是手账中的一页故事
- **心动收集**：用轻松的跑道式玩法承载情感叙事，降低操作门槛，强化情感代入
- **卡牌点亮**：将"集卡"与"恋爱进度"绑定，提供长线收集动力

### 1.3 目标用户
- 18-30岁女性用户
- 喜欢乙女向/恋爱模拟类内容
- 碎片化时间娱乐需求（单局约70秒）

## 二、游戏流程

```
开屏页 → 首页 → 选择约会角色 → 叙事引入（打字机效果）
                                        ↓
                                   跑道游戏（70秒）
                                        ↓
                                   结算 → 根据得分呈现不同结局文案
                                        ↓
                              卡牌星数累积 → 3★ 解锁专属卡牌
                                        ↓
                              集齐5张卡牌 → 全局终章
```

### 2.1 流程说明

| 阶段 | 说明 |
|------|------|
| **开屏页** | 品牌展示，支持自定义全屏图片，2.8秒后自动过渡 |
| **首页** | 游戏标题 + 开始约会 / 我的卡牌 两个入口 |
| **角色选择** | 展示5位男主头像、名字、当前卡牌进度，点击选择 |
| **叙事引入** | 逐字打字机效果呈现当前章节的约会场景描写 |
| **跑道游戏** | 核心玩法，70秒限时，左右滑动收集/躲避 |
| **结算页** | 根据得分评星（0-3★），逐字展示对应结局文案 |
| **卡牌系统** | 获得3★即累积1颗卡牌星，集满3颗星解锁角色专属卡牌 |
| **全局终章** | 集齐5张卡牌后触发终章叙事 |

## 三、核心玩法

### 3.1 跑道游戏规则

玩家控制女主角在 **5条赛道** 上左右移动，在 **70秒** 内尽可能多地收集爱意物品，同时躲避伤痛障碍。

| 参数 | 数值 | 说明 |
|------|------|------|
| 游戏时长 | 70秒 | 倒计时结束自动结算 |
| 赛道数量 | 5条 | 左右滑动/点击切换 |
| 生命值 | 5点 | 碰到障碍物-1，归零则提前结束 |
| 收集得分 | 100分/个 | 每收集一个爱意物品 |
| 物品下落速度 | 渐进加速 | 从0.7逐步提升至1.6 |
| 物品生成间隔 | 渐进缩短 | 从1200ms缩短至700ms |

### 3.2 物品类型

| 类型 | 图标 | 说明 | 效果 |
|------|------|------|------|
| **爱意物品** | 💗 | 角色专属的心动瞬间（如"替你挡酒""专属接送"等） | +100分 |
| **伤痛障碍** | 角色专属 | 角色专属的约会挫折（如"冷言拒绝""深夜加班"等） | -1生命 |

**障碍物图标按角色区分**：
- 霸总：🩶
- Idol：💙
- 医生：💙
- 甜品师：🧡
- 特警：🖤

### 3.3 评分与星级

| 星级 | 所需分数 | 结局质量 |
|------|----------|----------|
| ★★★ | ≥1500分 | 最甜蜜的心动结局 |
| ★★☆ | ≥900分 | 温暖但留有遗憾的结局 |
| ★☆☆ | ≥400分 | 淡淡的、若即若离的结局 |
| ☆☆☆ | <400分 | 错过彼此的遗憾结局 |

## 四、卡牌系统

### 4.1 卡牌收集规则
- 每位角色有 **3个章节**（第一章·初遇 → 第二章·心动 → 第三章·告白）
- 每个章节获得 **★★★（满星）** 即为该角色的卡牌累积 1颗星
- 累积 **3颗卡牌星** → 解锁该角色的专属收藏卡牌
- 集齐 **全部5张卡牌** → 触发全局终章

### 4.2 卡牌展示
- **恋爱手账（收藏页）**：以卡片墙形式展示5位角色的卡牌，已解锁卡牌显示角色图片，未解锁显示灰色锁定状态
- **卡牌详情**：点击已解锁卡牌，以 **翻转旋转动画** 放大展示，包含角色图片、名字、星级、经典台词
- **卡牌解锁**：解锁瞬间播放 **粒子绽放 + 卡牌翻转** 的华丽动效

## 六、视觉风格

### 6.1 整体风格
- **基调**：柔和浪漫、少女感、手账风
- **主色系**：统一粉紫色系（Pink-Purple Unified v2.1）
- **字体**：思源宋体（Noto Serif SC）用于标题与叙事，思源黑体（Noto Sans SC）用于界面元素
- **质感**：磨砂噪点纹理叠加、毛玻璃效果、柔和渐变

| 元素 | 色值 | 用途 |

## 七、技术实现

### 7.1 技术栈
- **纯前端 H5**：HTML5 + CSS3 + Vanilla JavaScript
- **渲染引擎**：Canvas 2D（游戏画面）+ DOM（界面层）
- **存储**：localStorage 本地持久化
- **无需后端**：零服务器依赖，可直接部署静态托管

### 7.2 性能优化
- 游戏背景使用 **离屏 Canvas 缓存**，避免每帧重绘静态元素
- HUD 信息（得分/生命/时间）仅在数值变化时更新 DOM
- 物品上限控制（屏幕最多6-8个物品），防止低端设备卡顿
- 使用 `requestAnimationFrame` 驱动游戏循环
